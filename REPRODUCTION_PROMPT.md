## 3. 3D Models & Assets

### Three.js / React Three Fiber usage
- Implement a particle field with approximately 1800 points for the hero experience.
- Implement a granule-sphere component composed of many small spheres arranged in a spherical cloud.
- Use a transmission crystal / icosahedron in the hero particle field.
- Animate the hero field through phases driven by a shared scroll progress value.

### 3D animation behavior
- Shared motion value should drive visual transformation through three phases:
  - Waste: muted earth tone, scattered cloud, slow rotation
  - Recycling: bright brand blue, faster rotation
  - Product: brand green, tighter cluster, settled motion
- The particle field should contract slightly as it approaches the product phase.
- The crystal and particle colors should lerp between phases.
- The granule sphere should rotate continuously and respond to selected color state.

### Lighting setup
- Ambient light for soft illumination
- Directional lights for highlight and depth
- Point light centered near the crystal or sphere
- Transparent materials and additive blending for the particle field

### Interactivity
- Granule sphere color selection buttons on the homepage should change the displayed sphere color.
- The homepage 3D preview should include color chips with accessible labels.

## 6. Deployment & Error-Free Setup

### Production readiness requirements
- Ensure the build succeeds with Vite.
- Ensure route generation works correctly.
- Ensure all internal links work with client-side routing.
- Ensure the favicon path is correct and local.
- Ensure the 404 and error pages render without breaking the app shell.
- Ensure the app is safe to deploy on Vercel or a similar static/server platform.

### Deployment notes
- Keep the app as a single-page app shell with route-based rendering and SSR support.
- Avoid hard-coded paths that break after deployment.
- Use relative or root-based internal links such as /products and /contact.
- Reference public assets via /Assets/...
- Keep the lead form submission behavior safe and graceful even if persistence is local-only.
- If a backend is not available, keep localStorage-based lead capture as a fallback and clearly isolate it so it can be swapped to a real API later.

### Build and run commands
- Install dependencies: bun install
- Start development server: bun dev
- Production build: bun run build
